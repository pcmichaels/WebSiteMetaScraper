				<header class="entry-header">
			
						<h1 class="entry-title">
				<a href="https://www.pmichaels.net/2020/04/11/moving-to-typescript-with-react/" rel="bookmark">Moving to Typescript with React</a>
			</h1>
										<div class="comments-link">
					<a href="https://www.pmichaels.net/2020/04/11/moving-to-typescript-with-react/#respond"><span class="leave-reply">Leave a reply</span></a>				</div><!-- .comments-link -->
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>Typescript gives you a number of nice little statically typed features, which Javascript is sadly lacking.  However, looking online, most of the tutorials and references use Javascript.  This is a handy little post to allow you (or me) to use Typescript; or to use Javascript, and then transition to Typescript.  Since you can use both in your project, you might find that migrating files one at a time allow you to gradually transition your code-base.</p>
<h1>File name</h1>
<p>When you create your <a href="https://www.pmichaels.net/2019/01/14/a-c-developers-guide-to-reactjs-part-1-create-run/">first React app</a>, you get a, by now, pretty standard SPA app.  The entire app is running Javascript.  To change one of the files over to Typescript, simply change the extension:</p>
<p><a href="https://www.pmichaels.net/wp-content/uploads/2020/03/typescript-react-1.png"><img src="https://www.pmichaels.net/wp-content/uploads/2020/03/typescript-react-1.png" alt="" width="294" height="113" class="alignnone size-full wp-image-4227" /></a></p>
<p>You now have your first Typescript component&#8230; Now the errors begin!</p>
<h1>Props</h1>
<blockquote><p>
Parameter &#8216;props&#8217; implicitly has an &#8216;any&#8217; type, but a better type may be inferred from usage.ts(7044)
</p></blockquote>
<p>This warning appears when you have the following constructor:</p>
<pre class="brush: jscript; title: ; notranslate" title="">constructor(props) {  
</pre>
<p>Now that you&#8217;ve moved from a dynamic to a statically* typed language, Typescript is simply suggesting that you may have a better type than the <em>Any**</em> type that is assigned by default.</p>
<p>The easiest, and best transition here is simply to tell Typescript the type; for example:</p>
<pre class="brush: jscript; title: ; notranslate" title="">interface IProps {
}

export class Home extends Component&lt;IProps&gt; {

  constructor(props: IProps) {    
</pre>
<p>If you actually have any properties, then you&#8217;ll need to include them in your interface; for example:</p>
<pre class="brush: jscript; title: ; notranslate" title="">interface IProps {
  myValue: boolean
}
</pre>
<h1>State</h1>
<p>In Javascript, you define state by usage in the constructor; for example:</p>
<pre class="brush: jscript; title: ; notranslate" title="">export class FetchData extends Component {
  static displayName = FetchData.name;
  constructor(props) {
    super(props);
    this.state = { forecasts: [], loading: true };
  }
</pre>
<p>Again, you need to define what the structure of state is:</p>
<pre class="brush: jscript; title: ; notranslate" title="">interface IState {
  forecasts: string[];
  loading: boolean;
}
</pre>
<p>Then, you can reference that in the constructor, and pass the type into the base component:</p>
<pre class="brush: jscript; title: ; notranslate" title="">export class Home extends Component&lt;IProps, IState&gt; {

  constructor(props: IProps, ) {    
    super(props);
    this.state = { 
      forecasts: [], 
      loading: true 
    };
  }
</pre>
<h1>External Interface Files</h1>
<p>One of the consequences of declaring these interfaces, is that you&#8217;ll quickly find that you need to reference the same interface from more than one place.  For example, you may read some data in one component, and then display it in another.  You could simply redeclare the interface in both places, but that feels dirty.</p>
<p>To get around this, you can simply declare a set of interfaces in a single file, or set of files.  In C#, I would declare a single interface per file, so I&#8217;m tempted to lean towards this approach:</p>
<p><a href="https://www.pmichaels.net/wp-content/uploads/2020/03/typescript-react-2.png"><img src="https://www.pmichaels.net/wp-content/uploads/2020/03/typescript-react-2.png" alt="" width="296" height="132" class="alignnone size-full wp-image-4226" /></a></p>
<p>This doesn&#8217;t necessarily apply to any interfaces that are relevant to only a single component (for example, the interface for the props &#8211; unless they are shared &#8211; feels like it&#8217;s perfectly happy in the same file as the component).</p>
<h1>Raising Events</h1>
<p>In the same way as you now have to define what props look like, when you <a href="https://www.pmichaels.net/2020/03/07/react-tips-5-raising-events/">raise an event</a>, you need to define what the event looks like; that is, you must tell Typescript the event type that you&#8217;re handling.</p>
<pre class="brush: jscript; title: ; notranslate" title="">interface SearchProps {
    searchText: string;
    searchAction: (e: React.MouseEvent&lt;HTMLButtonElement&gt;) =&gt; void;
}
function Search(props: SearchProps) {
    
    return (
        &lt;div&gt;
            &lt;label&gt;Search&lt;/label&gt;
            &lt;textarea&gt;{props.searchText}&lt;/textarea&gt;
            &lt;button onClick={props.searchAction}&gt;Search&lt;/button&gt;
        &lt;/div&gt;
    );
}
export default Search;

</pre>
<h1>External Libraries</h1>
<p>If you&#8217;re referencing an external library, you may not have the Typescript definitions; let&#8217;s take <a href="https://reactstrap.github.io/">reactstrap</a> for example:</p>
<blockquote><p>
Could not find a declaration file for module &#8216;reactstrap&#8217;. &#8216;c:/myproject/ClientApp/node_modules/reactstrap/dist/reactstrap.cjs.js&#8217; implicitly has an &#8216;any&#8217; type.</p>
<p>  Try `npm install @types/reactstrap` if it exists or add a new declaration (.d.ts) file containing `declare module &#8216;reactstrap&#8217;;`ts(7016)
</p></blockquote>
<p>You should be able to simply install this &#8211; which, in fact, it tells you to do in the error message:</p>
<pre class="brush: plain; title: ; notranslate" title="">npm install @types/reactstrap
</pre>
<h1>References</h1>
<p><a href="https://stackoverflow.com/questions/46987816/using-state-in-react-with-typescript/46987987">https://stackoverflow.com/questions/46987816/using-state-in-react-with-typescript/46987987</a></p>
<h1>Notes</h1>
<p> * I&#8217;m not convinced that Typescript qualifies as a true statically typed language, rather it provides <em>static typing</em> for a dynamically typed language.</p>
<p> ** <em>Any</em> is Typescript&#8217;s way of allowing you to revert to JavaScript, and just not type a specific variable; this is why I have doubts as to whether Typescript can be considered statically typed.</p>

			</h1>
										<div class="comments-link">
					<a href="https://www.pmichaels.net/2020/04/04/an-assembly-specified-in-the-application-dependencies-manifest-was-not-found-strange-errors-when-trying-to-create-an-ef-migration/#respond"><span class="leave-reply">Leave a reply</span></a>				</div><!-- .comments-link -->
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>This has been on my list to write up for some time, but this must be the third or fourth time I&#8217;ve encountered it &#8211; each time I get a vague sense that I&#8217;ve been here before.</p>
<h1>Setting the Scene</h1>
<p>It&#8217;s Friday afternoon, and you&#8217;re trying to create a migration, so you can check your code in and go home&#8230; or to the pub.  Disaster strikes, you get the following error:</p>
<blockquote><p>
  An assembly specified in the application dependencies manifest (MyProject.deps.json) was not found:<br />
    package: &#8216;Some.Package&#8217;, version: &#8216;1.0.0&#8217;<br />
    path: &#8216;Some.Package.dll&#8217;
</p></blockquote>
<p>You panic &#8211; there&#8217;s a pint of Doom Bar with your name on it, and it&#8217;s getting cold.  You do a quick search, and you find this post&#8230; actually, it doesn&#8217;t matter what I write here &#8211; you&#8217;ll just see the title and scroll down to the answer.  The Pigeon King is called Donald; there are 336 holes in your average golf ball; there&#8217;s nobody on the 37th floor, and nobody knows why.</p>
<h1>The Answer</h1>
<p>The answer is either that you have the incorrect project, or startup project.  You can change both of these manually, or simply specify them in the migration; for example:</p>
<pre class="brush: csharp; title: ; notranslate" title="">Add-Migration MigrationName -Context AppDbContext -StartupProject MyProject -Project MyProject.DataAccess
</pre>
<p>In my specific case, I had an Azure function set as the start-up project.</p>

			<p>In <a href="https://www.pmichaels.net/2020/02/25/an-adr-visual-studio-tool-part-6-visually-opening-files-and-changing-the-command-title/">my previous post</a> I continued with my little series on writing an extension for Visual Studio by completing the functionality to view existing ADRs within the solution.  You can see the first post <a href="https://www.pmichaels.net/2019/09/07/an-adr-visual-studio-tool-part-1-creating-a-visual-studio-extension-to-scrape-the-solution-and-list-all-items-including-solution-items-and-solution-folders/">here</a></p>
<p>In this post, we&#8217;ll cover the process of adding a command to the solution and project context menu.  Having completed the screen that will show existing ADRs, we now want to allow the user to right-click on a project or solution and select to add a new ADR:</p>
<p><a href="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-1.png"><img src="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-1.png" alt="" width="1072" height="944" class="alignnone size-full wp-image-4193" srcset="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-1.png 1072w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-1-300x264.png 300w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-1-1024x902.png 1024w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-1-768x676.png 768w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-1-624x549.png 624w" sizes="(max-width: 1072px) 100vw, 1072px" /></a></p>
<p>The first step is to add the command; fortunately, this is a pre-made template, so just select to add a new item (ironically), and pick Command:</p>
<p><a href="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-2.png"><img src="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-2.png" alt="" width="941" height="653" class="alignnone size-full wp-image-4194" srcset="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-2.png 941w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-2-300x208.png 300w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-2-768x533.png 768w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-2-624x433.png 624w" sizes="(max-width: 941px) 100vw, 941px" /></a></p>
<p>This will create you a menu item that will appear under the Tools menu by default, and will display a message box when selected:</p>
<p><a href="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-3.png"><img src="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-3.png" alt="" width="1382" height="742" class="alignnone size-full wp-image-4195" srcset="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-3.png 1382w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-3-300x161.png 300w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-3-1024x550.png 1024w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-3-768x412.png 768w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-3-624x335.png 624w" sizes="(max-width: 1382px) 100vw, 1382px" /></a></p>
<p>There are two things that we need to change about this command: the text (we don&#8217;t want it to read &#8220;Invoke AddAdrCommand&#8221;), and the location (we want it to be available from the right-click context menu of a project).  Both of those things are changed in the file <b>AdrPackage.vsct</b> (if your project is called Aardvark, this will be named AardvarkPackage.vsct).</p>
<p>If you have a look in that file, you&#8217;ll see something called MyMenuGroup; which is referenced in three places.  The first defines what it is:</p>
<p><a href="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-4.png"><img src="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-4.png" alt="" width="1219" height="164" class="alignnone size-full wp-image-4196" srcset="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-4.png 1219w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-4-300x40.png 300w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-4-1024x138.png 1024w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-4-768x103.png 768w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-4-624x84.png 624w" sizes="(max-width: 1219px) 100vw, 1219px" /></a></p>
<p>This is where you can change the command text (as I have above).</p>
<p>The second, where it is:</p>
<p><a href="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-5.png"><img src="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-5.png" alt="" width="1008" height="137" class="alignnone size-full wp-image-4197" srcset="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-5.png 1008w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-5-300x41.png 300w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-5-768x104.png 768w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-5-624x85.png 624w" sizes="(max-width: 1008px) 100vw, 1008px" /></a></p>
<p>This initially looks like this:</p>
<pre class="brush: plain; title: ; notranslate" title="">&lt;Parent guid=&quot;guidSHLMainMenu&quot; id=&quot;IDM_VS_MENU_TOOLS&quot; /&gt;
</pre>
<p>Which adds the menu to the top level Tools menu; change it to:</p>
<pre class="brush: plain; title: ; notranslate" title="">&lt;Parent guid=&quot;guidSHLMainMenu&quot; id=&quot;IDM_VS_CSCD_PROJECT_ADD&quot; /&gt;

<p>There are <a href="https://docs.microsoft.com/en-us/visualstudio/extensibility/internals/guids-and-ids-of-visual-studio-menus?view=vs-2017">other</a> options here.</p>
<p>The third is the <a href="https://docs.microsoft.com/en-us/visualstudio/extensibility/idsymbol-element?view=vs-2019">ID Symbol</a>:</p>
<p><a href="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-6.png"><img src="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-6.png" alt="" width="1158" height="115" class="alignnone size-full wp-image-4198" srcset="https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-6.png 1158w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-6-300x30.png 300w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-6-1024x102.png 1024w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-6-768x76.png 768w, https://www.pmichaels.net/wp-content/uploads/2020/03/adr-7-6-624x62.png 624w" sizes="(max-width: 1158px) 100vw, 1158px" /></a></p>
<p>If you decide to change the name of MyMenuGroup, you will need to do so in all three places above.</p>
<h1>Getting the Context of the Current Project</h1>
<p>Now that we&#8217;ve moved the menu to the context menu, we&#8217;ll need to find which project we&#8217;re in by accessing the DTE Service &#8211; this is retrieved by calling the command:</p>
<pre class="brush: csharp; title: ; notranslate" title="">            var dte = await package.GetServiceAsync(typeof(DTE)).ConfigureAwait(false) as DTE2;
</pre>
<p>You can then find out what&#8217;s selected by accessing the selected hierarchy:</p>
<pre class="brush: csharp; title: ; notranslate" title="">            UIHierarchy uih = (UIHierarchy)dte.Windows.Item(
                EnvDTE.Constants.vsWindowKindSolutionExplorer).Object;
            Array selectedItems = (Array)uih.SelectedItems;
</pre>
<p>Here&#8217;s the full code of the Execute method, to display the selected project:</p>
<pre class="brush: csharp; title: ; notranslate" title="">
        private async void Execute(object sender, EventArgs e)
        {
            await ThreadHelper.JoinableTaskFactory.SwitchToMainThreadAsync(package.DisposalToken);

            var dte = await package.GetServiceAsync(typeof(DTE)).ConfigureAwait(false) as DTE2;

            UIHierarchy uih = (UIHierarchy)dte.Windows.Item(
                EnvDTE.Constants.vsWindowKindSolutionExplorer).Object;
            Array selectedItems = (Array)uih.SelectedItems;
            foreach (UIHierarchyItem selectedItem in selectedItems)
            {
                // Show a message box to prove we were here
                VsShellUtilities.ShowMessageBox(
                    this.package,
                    selectedItem.Name,
                    &quot;Selected Project&quot;,
                    OLEMSGICON.OLEMSGICON_INFO,
                    OLEMSGBUTTON.OLEMSGBUTTON_OK,
                    OLEMSGDEFBUTTON.OLEMSGDEFBUTTON_FIRST);
            }
        }
</pre>
<p>We&#8217;re not far off now &#8211; there&#8217;s a few little issues, but the main thing that&#8217;s left is that we&#8217;re not actually adding anything &#8211; just displaying a message.  The next step is to get it to actually add a file, but we&#8217;ll come to that in the next post.</p>

			</h1>
										<div class="comments-link">
					<a href="https://www.pmichaels.net/2020/03/21/unit-testing-ef-core-how-to-invoke-the-contents-of-onmodelcreating/#comments">1 Reply</a>				</div><!-- .comments-link -->

				<div class="entry-content">
			<p>In <a href="https://www.pmichaels.net/2018/09/16/unit-testing-with-entity-framework-and-entity-framework-core-2-1/">this</a> post, I wrote about how you can test an EF Database, using an InMemory database.</p>
<p>I&#8217;m guessing a few people reading this will be thinking this is stating the bloody obvious, but it certainly wasn&#8217;t to me.  Imagine you have a DBContext with some seed data; for example:</p>
<pre class="brush: csharp; title: ; notranslate" title="">        protected override void OnModelCreating(ModelBuilder modelBuilder)
        {

            modelBuilder.Entity&lt;EntityType1&gt;().HasData(
                new EntityType1
                {
                    Id = &quot;1&quot;,
                    Name = &quot;Test1&quot;,
                    Description = &quot;Testing&quot;,
                    IsTest = true,
                    SomeNumber = 1
                },
</pre>
<p>In the above linked article, this seed data will never fire; however, simply calling:</p>
<pre class="brush: csharp; title: ; notranslate" title="">context.Database.EnsureCreated();
</pre>
<p>Once the context is created, will force the migrations to be run inside the in-memory instance, and you <em>should</em> end up with a system mirroring what you would see, should you run this migration against a physical database.</p>
<h1>Caveat Emptor</h1>
<p>When using this against EF Core 2.2 I found, what appeared to be, <a href="https://github.com/dotnet/efcore/issues/6872">this</a> issue.  The error being, while I was trying to insert a record, an error returned saying that an item with the same key exists on the table.  However, no such item should exist.  The linked article seems to imply that this relates to a bug with the insert for the in-memory database, and that it is resolved for EF Core 3.0.  I haven’t validated this, so it may, or may not work to upgrade to EF Core 3.0.  Please add a comment if you can validate or negate this.</p>

				<div class="entry-content">
			<p>Since I <a href="https://www.pmichaels.net/2018/02/04/using-unity-azure-functions/">first wrote about dependency injection in Azure Functions</a> things have moved on a bit.  These days, the Azure Functions natively* support DI.  In this post, I&#8217;ll cover, probably the most common, DI scenario: adding HttpClientFactory to your project.</p>
<p>I&#8217;ll assume that you have an Azure function, and that it looks something like this:</p>
<pre class="brush: csharp; title: ; notranslate" title="">    public static class Function1
    {
        [FunctionName(&quot;Function1&quot;)]
        public static async Task&lt;IActionResult&gt; Run(
            [HttpTrigger(AuthorizationLevel.Function, &quot;get&quot;, &quot;post&quot;, Route = null)] HttpRequest req,
            ILogger log)
        {
            log.LogInformation(&quot;C# HTTP trigger function processed a request.&quot;);

            string name = req.Query[&quot;name&quot;];

            string requestBody = await new StreamReader(req.Body).ReadToEndAsync();
            dynamic data = JsonConvert.DeserializeObject(requestBody);
            name = name ?? data?.name;

            return name != null
                ? (ActionResult)new OkObjectResult($&quot;Hello, {name}&quot;)
                : new BadRequestObjectResult(&quot;Please pass a name on the query string or in the request body&quot;);
        }
    }
</pre>
<p>I&#8217;ll assume that, because that&#8217;s the default HTTP Trigger function you get when you select to create a new function.</p>
<h1>NuGet Packages</h1>
<p>You&#8217;ll need a few NuGet packages; first, you&#8217;ll need:</p>
<pre class="brush: plain; title: ; notranslate" title="">Install-Package Microsoft.Extensions.Http
</pre>
<p>Which will allow you to use the HttpClientFactory.  You&#8217;ll also need some packages for the DI:</p>
<pre class="brush: plain; title: ; notranslate" title="">Install-Package Microsoft.Azure.Functions.Extensions
Install-Package Microsoft.NET.Sdk.Functions
</pre>
<h1>Startup</h1>
<p>If you create a new MVC project, you get a Startup class, which manages all your DI, etc.  So we&#8217;re going to create one here.  Create a Startup.cs class in the function app:</p>
<pre class="brush: csharp; title: ; notranslate" title="">    public class Startup : FunctionsStartup
    {
        public override void Configure(IFunctionsHostBuilder builder)
        {
            builder.Services.AddHttpClient();
        }
    }
</pre>
<p>The Configure method is a member of FunctionsStartup (ctrl-. to add the override).  You&#8217;ll also need to add the following line <em>outside</em> of the namespace:</p>
<pre class="brush: csharp; title: ; notranslate" title="">[assembly: FunctionsStartup(typeof(FullNamespace.Startup))]
</pre>
<p>Essentially, FullNamepsace here refers to the fully qualified <b>Startup</b> class in your project.  Without this line, nothing will be added to the IoC container.</p>
<p>The AddHttpClient call inside <em>Configure</em> adds HttpClientFactory to your IoC container.</p>
<h1>Azure Function</h1>
<p>If you have a look at the code above, you&#8217;ll notice the class is static.  We can&#8217;t have constructor injection into a static class (because we can&#8217;t have a constructor); let&#8217;s change that to an instance class:</p>
<pre class="brush: csharp; title: ; notranslate" title="">        public Function1(IHttpClientFactory httpClientFactory)
        {
            _httpClientFactory = httpClientFactory;
        }
</pre>
<p>You&#8217;ll also need to change the function itself from static to instance:</p>
<pre class="brush: csharp; title: ; notranslate" title="">        public async Task&lt;IActionResult&gt; Run(
</pre>
<p>That&#8217;s it &#8211; you can now reference HttpClientFactory from inside your function.</p>

				<div class="entry-content">
			<p>Raising an Event is an interesting concept is React: essentially, you pass a function into a component; and then, when the component handles an event, it simply calls the function handler.  This is referred to as &#8220;Raising&#8221; the event, because despite the event happening in the component, it is <em>dealt with</em> by a parent of that component.  In C#, you would refer to this as a delegate, and in C, this would be passing a pointer to a function.</p>
<p>Here&#8217;s a component:</p>
<pre class="brush: jscript; title: ; notranslate" title="">function MyComponent(props) {
    
    return (
        &lt;div&gt;
            &lt;label&gt;Change the text below&lt;/label&gt;
            &lt;input type='text' onChange={props.onChange} /&gt;
        &lt;/div&gt;
    );
}
export default MyComponent;
</pre>
<p>If you look at that code, you&#8217;ll see that the onChange event is handled by a function passed in, called onChange.  Here&#8217;s the calling component:</p>
<pre class="brush: jscript; title: ; notranslate" title="">export class Home extends Component {

  render () {
    return (
      &lt;div&gt;
        &lt;MyComponent onChange={this.onChange} /&gt;
      &lt;/div&gt;
    }
</pre>
<p>The onChange function in <em>Home</em> will be called whenever the <em>MyComponent</em> textbox is changed.</p>
<h1>Important Note</h1>
<p>Since this is React, one of the things that you may do in the onChange function is to update the state.  I have <a href="https://www.pmichaels.net/2019/07/27/react-tips-2-binding-an-event-handler/">previously written</a> on how to deal with this, or indeed any method which uses the <em>this</em> property.</p>

						<h1 class="entry-title">
				<a href="https://www.pmichaels.net/2020/02/29/add-entity-framework-core-to-an-existing-asp-net-core-project/" rel="bookmark">Add Entity Framework Core to an Existing Asp.Net Core Project</a>
			</h1>
										<div class="comments-link">
					<a href="https://www.pmichaels.net/2020/02/29/add-entity-framework-core-to-an-existing-asp-net-core-project/#respond"><span class="leave-reply">Leave a reply</span></a>				</div><!-- .comments-link -->

				<div class="entry-content">
			<p>This is one of those posts born of the fact that I&#8217;m constantly googling this, and coming up with videos and Microsoft docs &#8211; I just want a single, quick step by step guide; so here it is.</p>
<h1>Step One &#8211; NuGet packages</h1>
<p>If you&#8217;re using SQL Server, then you&#8217;ll need these packages (technically, you only need the first, unless you want to actually create or run a migration yourself):</p>
<pre class="brush: plain; title: ; notranslate" title="">Install-Package Microsoft.EntityFrameworkCore.SqlServer
Install-Package Microsoft.EntityFrameworkCore.Tools
</pre>
<h1>Step Two &#8211; Create DBContext (and entity classes if needed)</h1>
<p>The minimum DBContext looks like this:</p>
<pre class="brush: csharp; title: ; notranslate" title="">    public class MyDbContext : DbContext
    {

        public MyDbContext(DbContextOptions&lt;MyDbContext&gt; options)
            : base(options)
        {

        }

        public DbSet&lt;MyEntity&gt; MyEntities { get; set; }
        public DbSet&lt;MyOtherEntity&gt; MyOtherEntities { get; set; }
    }
</pre>
<p>You can feed your model directly into EF.  That&#8217;s much easier to do than it used to be, for example, any field called Id will automatically be inferred to be a Primary Key; however, at some point, you&#8217;re going to want to do something data specific to your model, and at that point, you lose the separation between data and business layers.  A nice way that I&#8217;ve found around this is to subclass your models:</p>
<pre class="brush: csharp; title: ; notranslate" title="">    public class MyEntity : MyModel
    {
    }
</pre>
<h1>Step Three &#8211; Startup.cs</h1>
<p>You&#8217;ll need to register the DbContext in your DI pipeline:</p>
<pre class="brush: csharp; title: ; notranslate" title="">        public void ConfigureServices(IServiceCollection services)
        {
            services.AddControllers();
            services.AddDbContext&lt;MyDbContext&gt;(a =&gt;
                a.UseSqlServer(Configuration.GetConnectionString(&quot;SqlConnectionString&quot;)));
        }
</pre>
<p>Your connection string goes in appsettings.json:</p>
<pre class="brush: jscript; title: ; notranslate" title="">{
  &quot;Logging&quot;: {
    &quot;LogLevel&quot;: {
      &quot;Default&quot;: &quot;Information&quot;,
      &quot;Microsoft&quot;: &quot;Warning&quot;,
      &quot;Microsoft.Hosting.Lifetime&quot;: &quot;Information&quot;
    }
  },
  &quot;AllowedHosts&quot;: &quot;*&quot;,
  &quot;ConnectionStrings&quot;: {
    &quot;SqlConnectionString&quot;: &quot;Server=(localdb)\\mssqllocaldb;Database=MyDatabase;Trusted_Connection=True;MultipleActiveResultSets=true&quot;
  }
}
</pre>
<h1>Step Four &#8211; Migration</h1>
<p>Since you&#8217;ve installed EF Tools, you can add a migration:</p>
<pre class="brush: plain; title: ; notranslate" title="">Add-Migration InitialMigration -StartupProject MySolution.MyProject
</pre>
<p>The other option is to switch on automatic migrations; although, I would personally advise against that, as you can quickly lose track of what it&#8217;s doing.</p>
<p>That&#8217;s it &#8211; once you run:</p>
<pre class="brush: plain; title: ; notranslate" title="">Update-Database
</pre>
<p>You should have a working EF instance.</p>
<h1>Step Five (optional) &#8211; Auto-Migration</h1>
<p>You may also wish to have the system upgrade itself to the latest version of the DB.  Whether you choose to do this depends on exactly how much control you need over the changes to your DB.  I&#8217;ve heard that some people create the migrations as SQL Scripts and hand them to a DBA.</p>
<p>Should you wish to have EF manage that for you, then you can do the following:</p>
<pre class="brush: csharp; title: ; notranslate" title="">
        public void Configure(IApplicationBuilder app, IWebHostEnvironment env)
        {
            if (env.IsDevelopment())
            {
                app.UseDeveloperExceptionPage();
            }

            UpdateDatabase(app);
</pre>
<p>. . .</p>
<pre class="brush: csharp; title: ; notranslate" title="">
private void UpdateDatabase(IApplicationBuilder app)
        {
            using var serviceScope = app.ApplicationServices
                .GetRequiredService&lt;IServiceScopeFactory&gt;()
                .CreateScope();

            using var context = serviceScope.ServiceProvider.GetService&lt;CourseSelectDbContext&gt;();

            context.Database.Migrate();            
        }
</pre>
<p>That&#8217;s it &#8211; now, next time I want to find this, I can search my own website!</p>

	<article id="post-4145" class="post-4145 post type-post status-publish format-standard hentry category-net-core category-asp-net-core category-c-2 tag-deserialize tag-jsonconvert tag-jsonserializer tag-newtonsoft tag-readasstringasync tag-serialisation tag-serialize tag-system-text-json">
				<header class="entry-header">
			
						<h1 class="entry-title">
				<a href="https://www.pmichaels.net/2020/02/22/my-object-wont-deserialise-using-system-text-json/" rel="bookmark">My object won&#8217;t deserialise using System.Text.Json</a>
			</h1>
										<div class="comments-link">
					<a href="https://www.pmichaels.net/2020/02/22/my-object-wont-deserialise-using-system-text-json/#comments">2 Replies</a>				</div><!-- .comments-link -->
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>I imagine this is a very common scenario, so I thought it worth documenting.</p>
<p>Let&#8217;s take the following controller method in a bog standard Asp.Net Core API:</p>
<pre class="brush: csharp; title: ; notranslate" title="">        public async Task&lt;DataContainer&gt; Get()
        {
            var data = Enumerable.Range(1, 5).Select(index =&gt; new MyData
            {
                Value1 = index,
                Value2 = &quot;some other value&quot;
            })
            .ToList();

            return new DataContainer()
            {
                Data = data
            };
        }
</pre>
<p>Here&#8217;s a method that consumes it:</p>
<pre class="brush: csharp; title: ; notranslate" title="">            var client = _clientFactory.CreateClient();
            var result = await client.GetAsync($&quot;https://localhost:22222/endpoint&quot;);

            using var responseStream = await result.Content.ReadAsStreamAsync();
            var data = await JsonSerializer.DeserializeAsync&lt;DataContainer&gt;(responseStream);
</pre>
<p>So, you&#8217;ve expect this to work, right?  It makes no difference what DataContainer looks like, because you&#8217;re serialising and deserialising using the same technology.  In fact, it just returns a correctly structured object, but every value will be null.</p>
<p>I&#8217;ll tell you what does work, is this:</p>
<pre class="brush: csharp; title: ; notranslate" title="">            var client = _clientFactory.CreateClient();
            var result = await client.GetAsync($&quot;https://localhost:22222/endpoint&quot;);

            var responseString = await result.Content.ReadAsStringAsync();
            var data = Newtonsoft.Json.JsonConvert.DeserializeObject&lt;DataContainer&gt;(responseString);             
</pre>
<p>If you look at the JSON, it looks kosher enough (again, the exact structure is irrelevant).  So, what&#8217;s the issue?</p>
<p>Interestingly, when the JSON is serialised, it is serialised into camelCase; so if you have a variable such as MyData, it will get serialised as myData.  Newtonsoft deals with this, because it&#8217;s a common use case in .Net; however, System.Text.Json.Deserialize* assumes that the casing will match the object!  </p>
<p>This is odd considering it <b>changes it on the way out!</b>!</p>
<p>Okay, so what&#8217;s the fix?</p>
<p>You just tell Deserialize* to use camelCase:</p>
<pre class="brush: csharp; title: ; notranslate" title="">            var client = _clientFactory.CreateClient();
            var result = await client.GetAsync($&quot;https://localhost:22222/endpoint&quot;);

            using var responseStream = await result.Content.ReadAsStreamAsync();

            var options = new JsonSerializerOptions()
            {
                PropertyNamingPolicy = JsonNamingPolicy.CamelCase
            };
            var data = await JsonSerializer.DeserializeAsync&lt;DataContainer&gt;(responseStream, options);

</pre>
<p>Annoying, eh?</p>

				<header class="entry-header">
			
						<h1 class="entry-title">
				<a href="https://www.pmichaels.net/2020/02/15/using-httpclientfactory/" rel="bookmark">Using HttpClientFactory</a>
			</h1>
										<div class="comments-link">
					<a href="https://www.pmichaels.net/2020/02/15/using-httpclientfactory/#respond"><span class="leave-reply">Leave a reply</span></a>				</div><!-- .comments-link -->
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>In .Net Framework (prior to .Net Core), HttpClient was something of a pain.  You essentially kept it around in a static variable.  If you didn&#8217;t, and used too many of them, you could end up issues such as socket exhaustion.</p>
<p>I think most people got around this by implementing their own version of a HttpClientFactory&#8230; but now you don&#8217;t need to.</p>
<p>In this post, I&#8217;m covering the simplest possible use case inside an Asp.Net Core application.  The articles linked at the bottom of this post go into much more detail.</p>
<h1>Usage</h1>
<p>The first step to using the HttpClientFactory is to add it to the IoC:</p>
<pre class="brush: csharp; title: ; notranslate" title="">        public void ConfigureServices(IServiceCollection services)
        {
            services.AddControllersWithViews();
            services.AddHttpClient();
        }
</pre>
<p>Then, simply inject it into your controller; for example:</p>
<pre class="brush: csharp; title: ; notranslate" title="">
        public HomeController(ILogger&lt;HomeController&gt; logger,
            IHttpClientFactory clientFactory)
        {
            _logger = logger;
            _clientFactory = clientFactory;
        }

</pre>
<p>Finally, you request a client like this:</p>
<pre class="brush: csharp; title: ; notranslate" title="">var client = _clientFactory.CreateClient();
</pre>
<p>Then you can use it as a normal HttpClient:</p>
<pre class="brush: csharp; title: ; notranslate" title="">var result = await client.GetAsync($&quot;https://localhost:22222/endpoint&quot;);
</pre>
<h1>References</h1>
<p><a href="https://docs.microsoft.com/en-us/aspnet/core/fundamentals/http-requests?view=aspnetcore-3.1">https://docs.microsoft.com/en-us/aspnet/core/fundamentals/http-requests?view=aspnetcore-3.1</a></p>
<p><a href="https://www.stevejgordon.co.uk/introduction-to-httpclientfactory-aspnetcore">https://www.stevejgordon.co.uk/introduction-to-httpclientfactory-aspnetcore</a></p>
